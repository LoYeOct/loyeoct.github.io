          # uni人脸考勤系统

          # 前言

          本次人脸考勤系统以uniAPP作为APP开发语言，主要运用uni本身的插件市场，活体检测插件（付费），以及高德地图的定位系统，采集到人脸图像后发送到服务端，还是用C#语言做服务端处理，调用百度的离线sdk采集功能保存人脸信息到人脸库，以后每次考勤对照着人脸库的人脸图片进行识别记录就行了。

          # 百度智能云配置

          登录百度智能云,在控制台找到“人脸识别”服务

          信息认证填完之后需要添加应用信息了，跟支付宝那边申请应用填写差不多Android和ios要分开申请，无非是一些名称，包名（好像是cmd命令会生成，google一下你就知道，做了有一段时间了，我忘记了嘿嘿嘿），应用场景和功能需求审核，审核通过后会获得固定的appid、appkey、secret_key，审核通过后百度这边的准备工作就完成了。

          # uniAPP配置

          uni这边是选择的直接去插件市场购买活体检测的SDK，插件地址，不打包发布的话还是能免费用的，可以直接去下载demo跑一下，采集好的图片是Base64格式的，传到服务端转化成相对路径图片。由于本项目还需要获取当前考勤打卡的地理位置，我选择的是高德地图的SDK，高德开放平台

          Android和ios一样，都是填写对应的安全码和包名即可生成对应的appkey

                        {"image_type", image_type2},
                        {"face_type", "LIVE"},
                        {"quality_control", "LOW"},
                        {"liveness_control", "NONE"},
