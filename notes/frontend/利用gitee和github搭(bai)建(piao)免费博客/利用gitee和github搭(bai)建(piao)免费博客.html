          # 利用gitee和github搭(bai)建(piao)免费博客

          说起搭(bai)建(piao)这个blog，对我这个.net后端来说，仿佛打开了新世界的大门，什么Vue,PowerShell,npm(“node包管理器”),npm wiki介绍，说实话还是挺有意思的，当然理解这些并不是本篇的重点，而且我还没怎么去了解，以后就机会再针对着写一下，目前只要会用轮子就行了。

          # gitee仓库

          需要在gitee上注册并建立一个私有仓库来存放我们的源文件,存到仓库方便无论在哪台电脑上都能下下来编译修改，前提得安装git(下载上传)和vscode(编写.md文件和终端运行编译)还有node.js(核心功能npm)。
          下载：如果没有在gitee上配置ssh，则下载用Https，建立个空文件,利用WIN+X+I组件打开PowerShell(又是一个新世界大门，用过之后不用cmd系列，有待钻研)，cd到建立的空文件，然后执行git clone 你的gitee仓库地址 ,下载成功如下图

          # github仓库

          github上需要建立的是一个公开的仓库（因为私有仓库需要付费,而且存放的都是.html的静态页面由github托管，算是github名下的子网，所以才需要gitee仓库分开存放源文件），仓库命名格式为：用户名.github.io

          # 通过vuepress将markdown文件编译成个人博客网站

          本篇重点，前两步都只是存放托管，方便个人使用更改，对会Vue等前端技术的朋友写这个只是花费一些时间成本，之后都只需要更改.md文件码字就行了（果然自己的博客还是得按自己随心所欲的修改风格和功能，要多舒服有多舒服），至于不会的前端的朋友，github设计了Pages功能，允许用户自定义项目首页，用来替代默认的源码列表。所以，github Pages可以被认为是用户编写的、托管在github上的静态网页。至于编译，可以在VScode自带的终端上运行，也可以在PowerShell上编译，下载node.js配套的npm 默认源是国外的，国内可以换上淘宝的源提升速度npm config set registry https://registry.npm.taobao.org 是否更换成功可以用 npm config get registry 查看

          1、前提记得CD到有package.json的配置文件那个文件夹 2、运行 npm i , 3、本地代码运行 npm run dev ，成功后会红框圈的就是本地预览的地址，属于热更新模式，即修改.md文件内容时会立即做对应的更新，基本上在本地运行没问题之后再更新到github上就行了

          4、上面所展示的只是从gitee仓下载下来的东西进行本地编译预览步骤，至于修改文件后的提交到github给别人预览，下面一张图就能清晰表明步骤了。哦对了，还有一处源码的gitee仓库需要同步，下载个git工具，绑定自己的VScode，输入账号密码后可以再左下角直接更新。

          # 结语

          勉强算是写完了吧，写完感觉写的不太好，自己也是跟着流程一边走一边写的，算是自我巩固了一次，一篇不太成熟的内容，有点写给自己看的意思，再次感谢Mercer-Lee的无私分享，知识是学不完的，无论什么方式，学到就是自己的宝贵财富，后面也会尽量恶补自身的知识缺陷。最后想起来这也搭(bai)建(piao)的博客有个缺点就是不翻墙的话浏览加载会慢很多，建议翻墙浏览，当然也不至于打不开，毕竟不花钱。如果想不翻墙浏览快一些（后面估计我也会配一个），就是在腾讯云买个域名，用腾讯的cdn加速，为了让自己的博客更正规更像个网站，嘿嘿嘿，.com后缀少不了，https协议少不了，国内公安部备案同样少不了。看个人需求吧，经过cdn加速的网站打开速度直接提升了N个档次，题外话说多了，好了，下一篇再见。
